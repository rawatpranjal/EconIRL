%% ============================================================
\section{Context and Problem}
%% ============================================================

Rust's NFXP algorithm \citep{rust1987} estimates DDC parameters by nesting the Bellman solve inside the likelihood maximization.
At each candidate $\theta$, the inner loop iterates the contraction mapping $V^{k+1} = T(V^k;\theta)$ until $\|V^{k+1} - V^k\|_\infty < \epsilon$.
When the discount factor $\beta$ is close to 1, the contraction rate $\beta$ approaches unity and convergence slows catastrophically.
At $\beta = 0.9999$, successive approximation (SA) requires
$\lceil \log(\epsilon)/\log(\beta) \rceil = \lceil \log(10^{-10})/\log(0.9999) \rceil = 230{,}258$
inner iterations \emph{per outer likelihood evaluation}.
With a fixed budget far below this threshold, the inner loop terminates early and the gradient passed to the outer optimizer is inaccurate, causing the parameter estimates to be biased or divergent.

\citet{suJudd2012} reframe estimation as a single constrained optimization problem.
The value function $V$ becomes an explicit decision variable and the Bellman equation $V = T(V;\theta)$ is imposed as a constraint.
This eliminates the inner fixed-point loop entirely.
The constraint is enforced by an augmented Lagrangian penalty, which grows across outer iterations until the Bellman residual falls below the feasibility tolerance.

%% ============================================================
\section{Model and Notation}
%% ============================================================

The agent observes state $s_t \in \calS$ with $|\calS| = n$, chooses action $a_t \in \calA$, and receives flow utility $u(s_t, a_t;\theta) + \epsilon_{t,a}$ where $\epsilon$ is Type I extreme value with scale $\sigma = 1$.
The state transitions according to $P(s'|s,a)$ and future payoffs are discounted at rate $\beta$.

\commonnotation

\noindent MPEC introduces $\lambda \in \R^n$ (Lagrange multipliers for the Bellman constraint) and $\rho > 0$ (augmented Lagrangian penalty weight, increasing across outer iterations).

%% ============================================================
\section{MPEC Formulation}
%% ============================================================

\paragraph{NFXP (nested).}
\begin{equation}
\label{eq:nfxp}
\max_\theta \; \mathcal{L}(\theta) \quad \text{where } V(\theta) \text{ solves } V = T(V;\theta) \text{ by contraction.}
\end{equation}

\paragraph{MPEC (joint).}
\begin{equation}
\label{eq:mpec}
\max_{\theta, V} \; \mathcal{L}(\theta, V) \quad \text{subject to} \quad V = T(V;\theta).
\end{equation}

The feasible set is identical: any $(\theta^*, V^*)$ satisfying the constraint achieves the same log-likelihood value as the NFXP solution.
The difference is computational: MPEC optimizes over $(\theta, V) \in \mathbb{R}^{p+n}$ jointly with the constraint enforced by a penalty, rather than eliminating $V$ via a slow fixed-point iteration.

\paragraph{Augmented Lagrangian.}
The constraint~\eqref{eq:mpec} is handled by the augmented Lagrangian
\begin{equation}
\label{eq:al}
\mathcal{L}_\rho(\theta, V, \lambda)
= -\mathcal{L}(\theta, V)
+ \lambda^\top [V - T(V;\theta)]
+ \tfrac{\rho}{2} \|V - T(V;\theta)\|^2.
\end{equation}
Each inner subproblem $\min_{\theta,V} \mathcal{L}_\rho$ is solved by L-BFGS-B.
The penalty $\rho$ grows by factor $\gamma_\rho$ each outer iteration until the Bellman residual $\|V - T(V;\theta)\|_\infty < \epsilon_c$.

%% ============================================================
\section{Algorithm}
%% ============================================================

\begin{algorithm}[h!]
\caption{MPEC via SQP \citep{suJudd2012}}
\label{alg:mpec}
\begin{algorithmic}[1]
\REQUIRE Data $\mathcal{D}$, transitions $P$, discount $\beta$, initial $\theta_0$
\ENSURE MLE parameters $\theta^*$, value function $V^*$

\STATE \textbf{Warm start:} solve $V_0 = T(V_0;\theta_0)$ by value iteration to get feasible $({\theta_0, V_0})$

\FOR{$k = 0, 1, \ldots, K_{\text{outer}}$}

  \STATE \textit{// Quadratic subproblem: linearize constraint around current iterate}
  \STATE Compute $\nabla_{(\theta,V)} \mathcal{L}$ and Jacobian $J = \nabla_{(\theta,V)} [V - T(V;\theta)]$
  \STATE Solve KKT system for SQP step $d^k$ and multipliers $\mu^k$:
         $\begin{bmatrix} B^k & J^\top \\ J & 0 \end{bmatrix}
          \begin{bmatrix} d^k \\ \mu^k \end{bmatrix} =
          \begin{bmatrix} -\nabla \mathcal{L} \\ -(V - T(V;\theta)) \end{bmatrix}$

  \STATE Line search on $\mathcal{L}(\theta + \alpha d^k_\theta,\, V + \alpha d^k_V)$; set $(\theta^{k+1}, V^{k+1}) \leftarrow (\theta^k, V^k) + \alpha d^k$

  \IF{$\|V^{k+1} - T(V^{k+1};\theta^{k+1})\|_\infty < \epsilon_c$}
    \STATE \textbf{break}
  \ENDIF

\ENDFOR

\STATE Compute standard errors via implicit function theorem at $(\theta^*, V^*)$
\RETURN $\theta^*$, $V^*$, standard errors

\end{algorithmic}
\end{algorithm}
\FloatBarrier

\noindent\textbf{Implementation notes.}
The warm start (line 1) initializes $V_0$ at the Bellman fixed point of the starting $\theta_0$, giving the SQP a feasible interior point.
$B^k$ is a BFGS approximation to the Lagrangian Hessian, updated each outer iteration.
Gradients of $\mathcal{L}$ and the constraint Jacobian $J$ are computed by JAX automatic differentiation.
In practice, the outer SQP uses scipy SLSQP with JAX-supplied gradients.

%% ============================================================
\section{Identification}
%% ============================================================

MPEC and NFXP solve the same MLE problem.
At convergence, the constraint $V^* = T(V^*;\theta^*)$ is satisfied exactly (up to feasibility tolerance), so $\mathcal{L}(\theta^*, V^*) = \mathcal{L}_{\text{NFXP}}(\theta^*)$ and both estimators recover the same population parameter vector.
Identification requires the same rank condition on the feature matrix $\phi(s,a)$ as NFXP: the matrix $\Phi$ formed by stacking $\phi(s,a)$ rows must have full column rank.
Scale is normalized by fixing $\sigma = 1$ (logit scale); level is normalized by resetting mileage to 0 on replacement, which anchors $RC$ in the same utility units as the operating cost $\theta_c$.
Standard errors are obtained by the observed information formula, evaluating the Hessian at $(\theta^*, V^*)$ with the implicit derivative $dV/d\theta = (I - \beta \nabla_V T)^{-1} \nabla_\theta T$ from the implicit function theorem.

\section{Simulation}
%% ============================================================

\input{mpec_results}

\textbf{Setting.}
We use the Rust bus engine with $n = \mpecNStates$ mileage bins, true parameters $\theta_c = 0.001$, $RC = 3.0$, and discount factor $\beta = \mpecBetaVal$.
Expert data: \mpecNObs{} observations (500 buses $\times$ 100 periods).
At $\beta = \mpecBetaVal$, successive approximation theoretically requires \mpecSANeeded{} inner iterations per outer likelihood call to reach tolerance $10^{-10}$; we cap NFXP-SA at \mpecSABudget{} to benchmark each method under the same setting.

\textbf{Results.}
All three estimators recover the same MLE: OC $= \mpecOC$ and RC $= \mpecRC$ (Bellman residual \mpecCV{}).
The difference is speed.
MPEC-SQP enforces the Bellman equation as an equality constraint in a single outer optimisation over $(\theta, V) \in \mathbb{R}^{p+n}$, running no inner iterations, and converges in \mpecMPECTime{}.
NFXP-SA, which solves the Bellman fixed point via contraction at each outer step, takes \mpecNFXPTime{} (\mpecSpeedup{} slower).
NFXP-NK, which replaces SA with Newton-Kantorovich inner iterations \citep{iskhakovRustSchjerning2016}, takes \mpecNKTime{} (\mpecNKSpeedup{} slower).
Eliminating the inner loop is the source of the speedup; MPEC does \emph{zero} inner Bellman iterations regardless of $\beta$.

Behavioral cloning achieves \mpecBCacc{} policy accuracy (LL $= \mpecBCLL$), confirming the data are informative but that BC cannot recover replacement costs without exploiting the MDP structure.

from econirl.environments.rust_bus import RustBusEnvironment
from econirl.estimation.mpec import MPECEstimator, MPECConfig
from econirl.preferences.linear import LinearUtility

env = RustBusEnvironment(num_mileage_bins=90, discount_factor=0.9999)
panel = env.generate_panel(n_individuals=500, n_periods=100)
utility = LinearUtility(env.feature_matrix, env.parameter_names)

mpec = MPECEstimator(MPECConfig(solver="sqp"))
result = mpec.estimate(panel, utility, env.problem_spec, env.transition_matrices)
print(result.parameters)   # [OC, RC] in ~0.6s
