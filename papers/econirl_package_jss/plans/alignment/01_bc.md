## Estimator: BC (Behavioral Cloning)
## Paper(s): Pomerleau (1991) "Efficient Training of Artificial Neural Networks for Autonomous Navigation," NeurIPS. The maximum-likelihood policy classifier idea generalizes far beyond Pomerleau, but his is the canonical citation in the IRL literature; the modern reference for BC as a baseline is Bain and Sammut (1995) and the imitation-learning textbooks.
## Code: `src/econirl/estimation/behavioral_cloning.py`

### Loss / objective

- Paper formula: maximize the log-likelihood of observed actions given states under a parametric or nonparametric policy class. For a tabular policy, the MLE is the empirical conditional choice frequency,

  ```
  pi_hat(a | s) = N(s, a) / N(s)
  ```

  where `N(s, a)` is the count of times action `a` was taken in state `s`. Add Laplace smoothing `alpha` if requested.

- Code implementation: `behavioral_cloning.py:96-107`. Counts state-action pairs with `counts.at[all_states, all_actions].add(1.0)`, adds `self._smoothing`, normalizes rows to one. The clip at `1e-10` on `row_sums` prevents division by zero for unvisited states.

- Match: **yes**, exact tabular MLE with optional Laplace smoothing. The smoothing default is `0.0` (pure MLE); the wrapper docstring documents `1.0` as the alternative.

### Gradient

- Paper formula: not applicable. The MLE has a closed-form solution.

- Code implementation: closed-form. No gradient computation.

- Match: **yes** (no gradient needed).

### Bellman / inner loop

- Paper formula: not applicable. BC ignores the MDP structure.

- Code implementation: `behavioral_cloning.py:82-86` explicitly documents that `utility` and `transitions` arguments are ignored. `value_function` is set to `jnp.zeros(n_states)` in the result.

- Match: **yes**.

### Identification assumptions

- Paper conditions: every state in the support of the data must be visited at least once for the corresponding row of `pi_hat` to be nonzero on at least one action. With Laplace smoothing, all rows are well-defined.

- Code enforcement: the `1e-10` clip ensures unvisited states get a uniform-style fallback only after the smoothing step. With `smoothing=0.0`, an unvisited state has `row_sums[s] = 1e-10`, making `policy[s] = counts[s] / 1e-10 = 0` everywhere — that is a degenerate row. The wrapper does not warn.

- Match: **approximate** — fine for non-degenerate panels, but the silent-degeneracy behavior is worth a wrapper-level warning.

### Hyperparameter defaults vs paper defaults

- `smoothing` default in code: `0.0`.
- Pomerleau original: not applicable (neural network MLE, no smoothing).
- Modern recommendation (Bain and Sammut 1995, and add-one rule): `smoothing=1.0` for small panels; `0.0` for large.

- Match: **acceptable** — pure MLE is the canonical default; the wrapper's docstring example sets `smoothing=1.0` so the user is at least pointed at the Laplace alternative.
